#  Backlog do Produto

O Backlog do Produto do **TrainerX64** representa uma lista priorizada de todas as funcionalidades, melhorias e requisitos necessários para o desenvolvimento do sistema de gestão de treinos voltado para personal trainers e seus alunos.

Esse backlog foi estruturado com base nas necessidades identificadas durante as etapas de Design Thinking, garantindo que cada item esteja alinhado com problemas reais enfrentados pelos usuários, como organização de treinos, acompanhamento de evolução e acessibilidade.

O gerenciamento do backlog foi realizado de forma colaborativa pela equipe, onde cada integrante contribuiu na criação, refinamento e priorização das histórias de usuário. A organização foi feita utilizando o **GitHub Projects**, permitindo visualizar o fluxo das atividades nas etapas de Backlog, Especificação, Revisão e Implementação.

---

## Como foi desenvolvido

- As histórias de usuário foram criadas seguindo o padrão:
  > Enquanto [tipo de usuário], desejo [funcionalidade], para [objetivo]

- Cada história foi associada a uma **persona**, garantindo rastreabilidade com as necessidades reais dos usuários.

- Foi realizada a **priorização** das histórias (Alta, Média, Baixa), considerando valor para o usuário e impacto no sistema.

- Parte das histórias foi detalhada com:
  - Critérios de Aceitação  
  - Regras de Negócio  

- O backlog foi organizado em colunas que representam o fluxo de evolução das histórias:
  - **Backlog** → histórias ainda não detalhadas  
  - **Especificação** → em refinamento  
  - **Revisão** → prontas para validação  
  - **Implementação** → prontas para desenvolvimento  

---

## Critérios de Aceitação e Regras de Negócio

Os **Critérios de Aceitação** definem as condições necessárias para que uma funcionalidade seja considerada concluída, garantindo alinhamento entre equipe e objetivos do sistema.

As **Regras de Negócio** estabelecem restrições e comportamentos do sistema, assegurando consistência e funcionamento adequado da aplicação.

---

## Exemplos de Histórias Especificadas

- **H01** — Enquanto personal, desejo cadastrar treinos padronizados para replicá-los rapidamente entre alunos com objetivos similares.  

- **H03** — Enquanto personal, desejo gerar relatórios automáticos de evolução dos alunos para apresentar resultados profissionais sem utilizar planilhas externas.  

- **H10** — Enquanto aluna, desejo acessar vídeos explicativos dos exercícios para garantir a execução correta.  

- **H16** — Enquanto usuário com baixa visão, desejo suporte a leitores de tela para navegar no sistema com autonomia.
